# portfolioMauricioobgo

Mauricio Obando's portfolio is now a **Python-only Flet static web app** built
with **Python 3.14**, **Flet**, and **uv**. The public UI is a dark
"Mauricio Cloud Console" experience that reads like an AI/cloud command center
instead of a generic portfolio page.

Deployment target: [https://mauricioobgo.github.io/portfolioMauricioobgo/](https://mauricioobgo.github.io/portfolioMauricioobgo/)

## Architecture
- `src/main.py` is the Flet static web entrypoint.
- `src/portfolio/` contains the Flet app, reusable components, sections, theme, and data loader.
- `src/assets/portfolio_content.json` is generated from the Python content pipeline and loaded by Flet at runtime.
- `src/assets/lottie/` contains local Lottie animation assets for the hero, project explorer, and AI assistant surfaces.
- `portfolio_app/data/*.yaml` stores curated content for profile, experience, certifications, and featured projects.
- `portfolio_app/generated/*.json` stores GitHub/profile refresh output, resume snapshots, AI context, and drift reports generated by automation.
- `portfolio_app/scripts/` owns GitHub sync, public resume sync, certification validation, cadence refresh, AI CLI access, and content building.

## Setup
```bash
uv python install 3.14
uv python pin 3.14
uv sync
```

## Local development
Refresh GitHub data directly:
```bash
uv run python -m portfolio_app.scripts.sync_github
```

Run cadence-aware refreshes:
```bash
uv run python -m portfolio_app.scripts.sync_data --scope weekly
uv run python -m portfolio_app.scripts.sync_data --scope monthly
```

Refresh only the public Drive CV snapshot and AI context:
```bash
uv run python -m portfolio_app.scripts.sync_resume
```

Build the generated content asset consumed by Flet:
```bash
uv run python -m portfolio_app.scripts.build_content
```

Ask the local/admin CV assistant:
```bash
uv run python -m portfolio_app.scripts.chat_cv --question "What production-grade systems does Mauricio build?"
```

Run the portfolio locally in the browser:
```bash
uv run flet run --web src/main.py
```

Create and test the production static build:
```bash
uv run flet build web --yes --base-url /portfolioMauricioobgo/ --route-url-strategy hash
uv run flet serve build/web
```

## Content pipeline
- `portfolio_app/scripts/sync_github.py` fetches GitHub profile and repository metadata through the GitHub API.
- `portfolio_app/scripts/sync_data.py` preserves the required cadence:
  - weekly for GitHub repositories
  - monthly for profile review, certifications review, and public CV refresh
- `portfolio_app/scripts/sync_resume.py` downloads the public Drive resume PDF, extracts text, writes `resume_snapshot.json`, produces `ai_context.json`, and emits a drift report.
- `portfolio_app/scripts/validate_links.py` ensures featured certifications include valid LinkedIn certifications URLs.
- `portfolio_app/scripts/build_content.py` merges curated YAML plus generated JSON into `src/assets/portfolio_content.json`.
- `portfolio_app/scripts/chat_cv.py` uses the OpenAI Python SDK and the Responses API for a local/admin-only CV assistant. The static site only exposes a UI stub and never ships an API key.

## Certifications strategy
- LinkedIn is **not scraped**.
- Certification cards are rendered from `portfolio_app/data/certifications.yaml`.
- Featured AWS certifications link to the LinkedIn certifications section, which is validated in CI.

## Resume and AI assistant strategy
- `portfolio_app/data/profile.yaml` carries both the human-friendly `resume_link` and the machine-friendly `resume_pdf_url`.
- Monthly automation refreshes additive artifacts only; it does **not** rewrite curated YAML from the resume.
- The public site exposes an "Ask Mauricio AI" section as a static UI stub.
- Real AI answers currently run through the local CLI using:
  - `portfolio_app/generated/ai_context.json`
  - `portfolio_app/generated/resume_snapshot.json`
  - `src/assets/portfolio_content.json`
- Set `OPENAI_API_KEY` locally before running the assistant CLI.

## Lottie assets
- Place animation files in `src/assets/lottie/`.
- The UI falls back gracefully if a Lottie asset cannot be loaded.
- Current assets are intentionally generic and safe for static hosting; you can swap them later without changing the Flet component structure.

## GitHub Pages deployment
- GitHub Actions builds the Flet static website and deploys it with the official Pages workflow.
- Pull requests to `main` run lint, tests, link validation, content build, and Flet web build.
- Pushes to `main`, manual runs, and weekly/monthly schedules rebuild the site and deploy via `actions/deploy-pages`.
- Repository Pages settings should use **GitHub Actions** as the deployment source.

## Updating content
- Profile: edit `portfolio_app/data/profile.yaml`
- Experience: edit `portfolio_app/data/experience.yaml`
- Certifications: edit `portfolio_app/data/certifications.yaml`
- Featured projects: edit `portfolio_app/data/featured_projects.yaml`
- Resume source URL and assistant prompts: edit `portfolio_app/data/profile.yaml`
- Refresh guidance: `docs-content-refresh.md`
