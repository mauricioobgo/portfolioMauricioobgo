# portfolioMauricioobgo

Mauricio Obando's portfolio now runs as a **Flutter Web frontend** backed by
**Python 3.14 + `uv`** tooling for curated content, GitHub refresh cadence, and
GitHub Pages deployment.

## Architecture
- `portfolio_app/` keeps the Python content and sync pipeline.
- `portfolio_app/data/*.yaml` stores curated profile, experience, project, and certification content.
- `portfolio_app/generated/` is refreshed by the weekly and monthly sync commands.
- `frontend/` is the Flutter Web app that renders the public site.
- `frontend/assets/data/portfolio_content.json` is generated by Python and consumed by Flutter.

## Setup
```bash
uv python install 3.14
uv venv --python 3.14
uv sync
```

Flutter is required for the frontend:
```bash
flutter --version
```

## Local development
Build the frontend content payload from the Python source of truth:
```bash
uv run python -m portfolio_app.scripts.build_frontend_content
```

Run the data sync when you want fresh GitHub/profile payloads:
```bash
uv run python -m portfolio_app.scripts.sync_data --scope weekly
uv run python -m portfolio_app.scripts.sync_data --scope monthly
```

Run the Flutter web app:
```bash
cd frontend
flutter pub get
flutter run -d chrome --web-port 3000
```

Create the production web build used by GitHub Pages:
```bash
cd frontend
flutter build web --release --base-href /portfolioMauricioobgo/ --pwa-strategy=none
```

## Quality commands
Python:
```bash
uvx --from ruff ruff check .
uvx --from ruff ruff format --check .
uvx --with . --from pytest python -m pytest -q
```

Flutter:
```bash
cd frontend
dart format --set-exit-if-changed .
flutter analyze
flutter test
```

## Refresh cadence
- GitHub repositories/project showcase: weekly
- LinkedIn/profile details: monthly
- Certifications: monthly

Operational checklist: `docs-content-refresh.md`

## GitHub Pages deployment
- Pull requests to `main` run Python checks plus Flutter format/analyze/test/build.
- Pushes to `main` run the full sync, rebuild the content payload, build Flutter Web, and publish `frontend/build/web` to the `gh-pages` branch.
- Scheduled runs keep the weekly and monthly sync cadence separate before rebuilding and redeploying the site.
- Repository Pages settings should stay on `Deploy from a branch`, source `gh-pages`, folder `/ (root)`.
